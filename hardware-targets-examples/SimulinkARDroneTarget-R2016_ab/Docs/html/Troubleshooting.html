Troubleshooting

This section provides awnsers or tips for troubleshooting any issues you encounter.

I uploaded a flight model to the AR Drone 2.0 but It doesn't fly, why is this?

The flight models contain safety checks which prohibit the drone from flying if:

  • The battery voltage is too low
  • The drone is positioned at an unsafe angle (if the drone is calibrated wrong it might falsely think it is in an unsafe orientation)

Why am I getting a "ExtTargetPktPending() call failed" error?

There are multiple reasons this can occur:

  • Too much information is being send back to Simulink causing the drone lose connection. Reduce the amount of scopes or other real time sources or use a rate transition block to reduce their sampling rate.
  • The wireless communication between the AR Drone and PC ran into an issue, re-building and reconnecting will allow you to run the model again.

Why am I getting the "Output argument "out" (and maybe others) not assigned during call to "coder.make.Builder/getCustomToolchainOptionMacros"." error?

  • This error can result from having multiple copies of the AR Drone 2.0 Target set up on the same MATLAB installation. Each one will register the Code Sourcery compiler, updating the path. If you delete the AR Drone 2.0 Target which was configured last, or if you deleted the registration files, you will need to run install_script.m to update the third party registration.

Why does initialization of the project path fail during the Simulink project startup?

  • The path is set up when the Simulink project is started and removed when the project is closed. If you close MATLAB without closing the Simulink project first the shutdown procedure is not ran meaning the path is not updated. While this will show as a failure to initialize the project path the path will be correct after opening the project.

Why does connecting to the AR Drone result in the following error: The model that you are trying to run in External mode does not match the application running on your target. Rebuild the model, run it on your target, and start External mode simulation again.

  • The problem is likely with the AR Drone, you will need to reboot the AR Drone (for which you can find a shortcut in the simulink project), reconnect to it and rebuilding the model should work

Why does connecting to the AR Drone result in the following error: The first input to get_param must be of type 'double', 'char' or 'cell'.

  • There is an issue with getting the data from the configuration set. Make sure the configuration set is set properly. If it still persists try restarting the Simulink project.
