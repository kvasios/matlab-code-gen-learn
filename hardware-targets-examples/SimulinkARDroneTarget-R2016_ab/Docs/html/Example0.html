Example: Configuring a model for the AR Drone 2.0

The AR Drone 2.0 Target comes with pre configured models that will run on the AR Drone 2.0 out of the box. If you wish to enable AR Drone 2.0 code generation for your own models, you can do so by following the steps in this example.

Configuring the model

Use the following steps to enable automatic code generation for your Simulink model:

  • Open your model: Open a new or existing Simulink model. Make sure that the components which you include support code generation (which is true for most Simulink blocks).
  • Link the model to the data dictionary: Within your Simulink model navigate to File >> Model Properties >> Link to data dictionary... . To learn more about data dictionaries, see the Data dictionaries page in the concepts section of this documentation.
  • Select the ArDroneDataDictionary: Click on Data Dictionary >> Browse... and then select \AR_Drone_Target\ArDroneDataDictionary.sldd
  • Accept the dialogs: Press OK followed by Yes to continue and finally press OK again after the linking completes.
  • Choose your simulation mode: You can run code in external or normal mode, see the Simulation modes concept page for more information on which to use. Enter the desired mode in the Simulink model area shown:

  • Open the model explorer: The data dictionary contains two configuration sets. To set one we need to open the model explorer using the button in the model or by pressing control + H.
  • Add the configuration set by reference: Within the model explorer, select the model you wish to generate code for, then press the Add reference button which will open up the following menu:

  • Select the desired configuration set: Choose AR_Drone_ConfigSet if you want to use external mode, for normal mode select AR_Drone_ConfigSet_NOEXT
  • Activate the reference configuration set: With two configuration sets set (the default one and the newly selected referenced configuration set), the correct configuration set needs to be activated. Right click the new Reference and select activate such that it appears as in the below figure.

Your model is now all set for automatic code generation for the AR Drone 2.0, remember to save your model to make these changes persistent.
