Concepts

The here included subsections explain the global concepts involved in the creation and use of the AR Drone 2.0 Target.

Ar Drone 2.0 hardware

The AR Drone 2.0 contains various sensors, processing boards and motors. In order to be able to fly the AR Drone 2.0 it is important to understand the hardware capabilities and limitations.

Integrating drivers

Reading the sensor data, video streams or sending control inputs to the motors of the AR Drone 2.0 requires interfacing between Simulink blocks and the drone operating system. This section explains how c code drivers can be implemented as Simulink blocks.

Code generation

Simulink allows for easy prototyping using a graphical interface to construct models. These models have to be translated into c code that is compatible with the AR Drone 2.0. This section explains the workflow that is executed by the Embedded Coder when you press the "Build" button to get the model running on the AR Drone 2.0.

Configuration parameters

The parameters in a configuration set determine the type of solver used, import and export settings, and other values that determine how the model runs. The AR Drone 2.0 Target comes with 2 specific configuration sets which are needed to run models on the AR Drone and explained in this section.

Data dictionaries

The AR Drone 2.0 Target uses a Data dictionary to save some key data. By using a data dictionary the essential settings and parameters required for getting a model running on the AR Drone 2.0 are available in a single location. This section explains how to access and apply the data dictionary provided with the AR Drone 2.0 Target.

Simulation modes

The AR Drone 2.0 Target lets you run models on the AR Drone 2.0 in external or normal mode. This section explains the differences between the two.
